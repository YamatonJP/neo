# 要件定義書テンプレート

`docs/features/<slug>/requirements.md` は以下の構成で書く。該当なしのセクションは「なし」と明記して残す(書き忘れと区別するため)。

```markdown
# 要件定義: <機能名>

- ステータス: draft | approved
- 作成日: <YYYY-MM-DD>
- 依頼の原文: <ユーザーの依頼をそのまま>

## 背景・目的

なぜこの機能が必要か。誰のどんな課題を解決するか。

## スコープ

### やること
- ...

### やらないこと
- ...(意図的に外したものを明示)

## ユーザーストーリー

- <役割> として、<目的> のために、<操作> したい

## 受け入れ基準

検証可能な形で列挙する。「〜できる」ではなく「〜したとき〜になる」。

- [ ] <条件> のとき <期待結果>
- [ ] ...

## 非機能要件

性能・セキュリティ・互換性・アクセシビリティ・運用(失敗時の検知方法)・リリース戦略(フラグ/段階公開)など。なければ「なし」。

## 制約

期限、依存ライブラリの制限、既存仕様との整合など。

## 未決事項

議論の結果まだ決まっていないこと。**空欄にせず、決まっていないことを正直に書く。**

## 議論ログ

critic との議論で出た主要な論点と決定理由を簡潔に(後から「なぜこうしたか」を追えるように)。

| 論点 | 決定 | 理由 |
| :-- | :-- | :-- |
```
